Funcionalidades

Todo lo que necesita tu cafetería

Cinco módulos interconectados que cubren el ciclo operativo completo, desde la venta hasta el análisis de resultados.

🔐

Autenticación segura

Acceso con usuario y contraseña. Token JWT de 8 horas con verificación de sucursal activa y control de sesión persistente.

🧾

Registro de ventas

Selección de productos del catálogo, cálculo automático del total, descuento de stock y asignación de puntos al cliente en una sola operación.

📦

Control de inventario

Stock por sucursal con alertas automáticas al alcanzar el umbral mínimo configurable. Actualizaciones manuales por el gerente.

👥

Clientes y fidelización

Base de clientes con historial de compras y sistema de puntos acumulables. Canje de puntos en ventas posteriores.

📊

Reportes por sucursal

Totales de ventas, productos más vendidos y comparativo entre sucursales filtrado por periodo. Acceso exclusivo para gerente y super admin.

Flujo de trabajo

¿Cómo funciona?

Tres pasos simples para que tu equipo opere con total claridad y control desde el primer día.

1

Configura tu negocio

El gerente registra las sucursales, carga el catálogo de productos con precios y stock mínimo, y asigna los vendedores a cada punto de venta.

2

Opera en tiempo real

El vendedor registra ventas seleccionando productos del catálogo. El sistema descuenta stock automáticamente, acumula puntos al cliente y emite el resumen de la transacción.

3

Analiza y decide

El gerente y el super admin consultan reportes por sucursal y periodo: ventas totales, productos más vendidos y alertas de reposición de stock.

Control de acceso

Tres roles, responsabilidades claras

Cada usuario accede únicamente a lo que su rol permite. El aislamiento por negocio garantiza que cada tenant opera de forma independiente.

Super Admin

Super Administrador

Acceso global a la plataforma
  • → Gestiona todos los negocios registrados
  • → Crea y bloquea sucursales
  • → Administra usuarios de cualquier negocio
  • → Accede a reportes consolidados
Gerente

Gerente

Gestión del negocio propio
  • → Administra sucursales de su negocio
  • → Configura catálogo y stock mínimo
  • → Gestiona el equipo de vendedores
  • → Consulta reportes y comparativos
Vendedor

Vendedor

Operación en punto de venta
  • → Registra ventas desde su sucursal
  • → Asocia clientes y acumula puntos
  • → Consulta stock disponible
  • → Accede solo a su sucursal activa

Caso de uso · CU-01

Registra tu cafetería en DXN SaaS

Este formulario simula el caso de uso principal del sistema: el registro de un nuevo negocio. Cada campo está mapeado a las entidades del Diagrama Entidad-Relación (DER).

Datos del negocio → negocios
→ negocios.nombre (TEXT, NOT NULL)
→ negocios.email_contacto (TEXT)
Datos del gerente → usuarios
→ usuarios.nombre (TEXT, NOT NULL)
→ usuarios.email (TEXT, NOT NULL, UNIQUE)
→ usuarios.usuario (TEXT, NOT NULL, UNIQUE)
→ usuarios.rol (TEXT, DEFAULT 'vendedor')
Sucursal principal → sucursales
→ sucursales.nombre (TEXT, NOT NULL)
→ sucursales.direccion (TEXT)
✅

Registro simulado con éxito

Los datos ingresados se mapearían a las tablas negocios, usuarios y sucursales de la base de datos PostgreSQL del sistema DXN SaaS.

Mapeo Interfaz → Base de Datos

Cada campo del formulario corresponde a un atributo real del DER del sistema. Al enviar, se simula la inserción de registros en tres entidades relacionadas.

  • Nombre negocio → negocios.nombre TEXT
  • Email contacto → negocios.email_contacto TEXT
  • Nombre gerente → usuarios.nombre TEXT
  • Email gerente → usuarios.email TEXT UNIQUE
  • Usuario → usuarios.usuario TEXT UNIQUE
  • Rol → usuarios.rol TEXT
  • Nombre sucursal → sucursales.nombre TEXT
  • Dirección → sucursales.direccion TEXT
Claves generadas automáticamente: Los campos id (UUID PK), negocio_id (FK), contrasena_hash y creado_en son generados por el backend y la base de datos — no se capturan en el formulario.
